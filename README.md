# Multi-Label Classification Pipeline

Репозиторий содержит полное решение задачи многоклассовой классификации (41 таргет). Пайплайн построен так, чтобы минимизировать утечки данных, обеспечить воспроизводимость и выжать максимум из ансамблирования. Основной стек: Python, Polars, CatBoost, LightGBM, PyTorch.

Здесь нет волшебства, только инженерия данных и аккуратная валидация. Ниже подробно расписано, как запускается процесс и почему были приняты те или иные технические решения.

---

## 🔄 Как устроен процесс (Pipeline Flow)

Мы не обучаем модели хаотично. Есть четкая последовательность этапов, где выход одного скрипта становится входом для другого. Это позволяет останавливаться на любом этапе и продолжать работу без потери прогресса.

### 1. Обработка данных (`scripts/00_full_data_processing.py`)
Все начинается здесь. Мы не просто грузим CSV, мы строим фундамент.
*   **Polars вместо Pandas:** Для скорости и экономии памяти. На больших данных это дает выигрыш в 2-3 раза.
*   **Защита от утечек (Leakage):** Это критичный момент. Флаги пропусков (Null Flags) создаются **без использования таргета**. Мы смотрим только на частоту пропусков, чтобы не слить информацию из будущего в признаки.
*   **Статистики строки до импутации:** `row_mean`, `row_std` и другие статистики считаются по "грязным" данным (с NaN). Сам факт пропуска часто информативнее, чем заполненное медианой значение.
*   **Проверка стабильности:** Скрипт сравнивает распределения признаков в train и test. Если признак сильно "плывет" (shift > 15%), он помечается. Это защита от переобучения на артефактах сбора данных.
*   **Sparse PCA для пропусков:** Паттерны пропусков сжимаются через SVD. Это добавляет 20 признаков, которые описывают структуру missing values, экономя при этом память.

### 2. Генерация фолдов (`scripts/01_generate_folds.py`)
Мы не генерируем разбиения внутри обучения. Это сделано один раз и навсегда.
*   **Разделение ответственности:** Скрипт сохраняет только индексы строк и ID клиентов. Сами данные не дублируются.
*   **Универсальные пути:** Скрипт сам находит корень проекта (по наличию `.git`), поэтому пути работают одинаково на Windows, Linux и macOS без хардкода абсолютных путей.
*   **Метаданные:** В конфиг фолдов записывается список категориальных признаков и таргетов. Дальнейшие скрипты подхватывают этот контекст автоматически.

### 3. Stage 1: Proxy Training (`scripts/02_stage1_proxy_training.py`)
На этом этапе мы готовим "бензин" для второго уровня.
*   **Multi-Model OOF:** Обучаем CatBoost, LightGBM и Neural Network на фолдах. Нам нужны не сами модели, а их Out-of-Fold предсказания.
*   **Калибровка:** Сырые вероятности моделей часто занижены или завышены. Мы применяем **Isotonic Regression**, чтобы вероятность 0.8 означала реальные 80% шансов. Это критично для корректного блендинга.
*   **Чекпоинты:** Если обучение прервалось, скрипт проверит, какие фолды уже обучены, и продолжит со следующего. Не нужно тратить время на повторный расчет.

### 4. Stage 2: Validation & Ensemble (`scripts/03_stage2_validation.py`)
Здесь происходит основная магия улучшения метрики.
*   **Мета-признаки из корреляций:** Мы знаем, что таргеты коррелируют друг с другом. Предсказание таргета А становится признаком для таргета Б, если они связаны. Модели начинают "общаться" друг с другом.
*   **Отбор признаков:** Оставляем только топ-1500 признаков по важности. Это убирает шум и ускоряет инференс.
*   **Per-Target Оптимизация весов:** Это ключевая фишка. Мы не используем одни веса ансамбля для всех 41 таргета. Для каждого таргета отдельно (через Differential Evolution) подбирается вклад каждой модели. Где нейросеть ошибается — её вес обнуляется, где сильна — увеличивается.

### 5. Инференс и Сабмит (`scripts/04_inference_submission.py`)
Финальная сборка.
*   **Консистентность:** Мета-признаки для теста генерируются точно так же, как для валидации, используя предсказания Stage 1 моделей. Никакого рассинхрона.
*   **Фиксация типов:** Платформы часто отклоняют файлы из-за типов данных. Скрипт автоматически приводит `customer_id` к int32, а предсказания к float64 перед сохранением.
*   **Блендинг:** Применяется карта весов, сохраненная на Стадии 2. Это гарантирует, что сабмит точно соответствует лучшей валидационной стратегии.

## 📂 Описание файлов и утилит

Здесь нет лишних файлов. Каждый скрипт закрывает конкретную задачу в конвейере.

### `scripts/00_full_data_processing.py`
Гибридный процессор данных на Polars, реализующий защиту от утечек через создание флагов пропусков без использования таргета, расчет статистик строки до импутации и автоматическую проверку стабильности распределений признаков между train и test выборками для предотвращения переобучения.

### `scripts/01_generate_folds.py`
Генератор кросс-валидации, который создает воспроизводимые фолды один раз и сохраняет их на диск вместе с метаданными, поддерживая универсальные пути для работы в любой ОС и позволяя разделять процесс разбиения данных от процесса обучения моделей.

### `scripts/02_stage1_proxy_training.py`
Движок обучения базовых моделей (CatBoost, LGBM, NN) для получения OOF предсказаний, включающий систему чекпоинтов для пропуска уже обуч
